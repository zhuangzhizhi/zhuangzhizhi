git的四种对象(blob tree commit tag)

四种对象用一个39位的hash值(2(文件夹名字)+38(文件名))来作为唯一标记

blob

当修改，新增文件被提交到暂存区stage时(创建文件夹时提交不了)，.git/objects/ 目录下创建修改或者新增的增量记录，一个文件一个文件夹，当commit到仓库时，创建一个commit对象，里面包含着一个tree对象，tree对象包含着这次的提交内容，提交内容若有文件夹则会另外创建一个tree对象包含着这个目录下的提交内容(以上的commit对象，tree对象，blob对象都由一个hash值来唯一标识，若有两个文件内容相同的时候，只会创建一个blob对象，即blob对象不存储提交的文件信息，只存储文件内容)
